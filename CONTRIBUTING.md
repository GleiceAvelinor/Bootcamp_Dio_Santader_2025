# Contribuindo para Bootcamp DIO Santander 2025

Obrigado por considerar contribuir para este projeto! 🎉

## Como Contribuir

### 1. Reportar Bugs
Se encontrar um bug, abra uma issue descrevendo:
- O que você estava tentando fazer
- O comportamento esperado
- O comportamento real
- Passos para reproduzir o problema

### 2. Sugerir Melhorias
Para sugestões de novas features:
- Descreva o caso de uso
- Explique o benefício
- Cite exemplos se possível

### 3. Enviar Pull Requests

1. Faça um fork do repositório
2. Crie uma branch para sua feature (`git checkout -b feature/nova-feature`)
3. Commit suas mudanças com mensagens claras
4. Push para a branch (`git push origin feature/nova-feature`)
5. Abra um Pull Request com descrição detalhada

### Diretrizes de Código

- Siga PEP 8 para Python
- Adicione docstrings em funções
- Inclua testes para novas funcionalidades
- Atualize o README se necessário

### Processo de Review

Todos os PRs serão revisados. Pode ser solicitado:
- Ajustes de código
- Testes adicionais
- Documentação extra

## Código de Conduta

Esperamos um comportamento respeitoso e profissional de todos.
